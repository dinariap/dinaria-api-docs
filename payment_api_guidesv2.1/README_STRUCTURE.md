
# Documentation Structure

This version introduces a new structure designed for both:

- Developers integrating the API
- Product teams understanding money movement

## Model

Money In → Accounts → Money Out

Participants:

Customer → sender
Beneficiary → receiver

FX connects currencies.
Cash is treated as a payout rail.
