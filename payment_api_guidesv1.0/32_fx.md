# FX Handling

FX is required when the payout **source currency** and **destination currency** differ.

Dinaria supports two modes:

- **Quote mode:** create a quote, then execute the payout using `fxQuoteId` (locks the rate).
- **Auto-quote mode:** set `autoQuote=true` and Dinaria will quote during payout creation.

## amountType

`amountType` controls what value is fixed:

| amountType | Meaning | Typical use |
|---|---|---|
| `source` (default) | The debit amount is fixed in the source currency. | You know how much you want to spend. |
| `destination` | The receive amount is fixed in the destination currency. | Remittance / payroll: beneficiary must receive an exact amount. |

## Common patterns

### Lock rate (recommended for production)

1) `POST /fx/quotes`
2) `POST /payouts` with `fxQuoteId`

### Auto-quote (simplest)

`POST /payouts` with:

- `autoQuote=true`
- `amount`, `currency`, and optionally `amountType`

If currencies mismatch and you do **not** provide `fxQuoteId` or `autoQuote=true`, Dinaria returns `FX_REQUIRED`.
